name: migrations
description: Write a migration to change the database schema
---

Migrations are managed using Alembic (for SQLAlchemy), and live in `src/jobwatch/migrations/`.

Before creating a migration, edit the SQLAlchemy model in src/jobwatch/models.py.

To generate a new migration, use:

```sh
alembic revision --autogenerate --head heads --message useful_migration_label
```

As much as possible, the autogenerated code should be used.

To test, run

```sh
alembic upgrade heads
alembic downgrade <revision>
```

Upgrade paths must be well tested, but certain hard-to-downgrade changes are allowed to be skipped (ie, un-adding an
enum member), provided the upgrade is idempotent.

Note that unit tests do not use alembic migrations - tables are created directly, using `Base.metadata.create_all()` each run.

---
name: migrations
description: Write a migration to change the database schema
---

Migrations are managed with Alembic and live in `src/jobwatch/migrations/`.

Alembic loads the app config, so `ENVIRONMENT` must be set, and autogenerate
compares against the live database — Postgres must be running
(`docker compose up -d`) and migrated to heads.

1. Edit the SQLAlchemy models in `src/jobwatch/models.py`.
2. Generate the migration:

```sh
ENVIRONMENT=development uv run alembic revision --autogenerate --head heads --message useful_migration_label
```

As much as possible, the autogenerated code should be used.

To test, run

```sh
ENVIRONMENT=development uv run alembic upgrade heads
ENVIRONMENT=development uv run alembic downgrade <revision>
```

Upgrade paths must be well tested, but certain hard-to-downgrade changes are allowed to be skipped (ie, un-adding an
enum member), provided the upgrade is idempotent.

Note that unit tests do not use alembic migrations - conftest.py creates tables directly, using `Base.metadata.create_all()` each run.
